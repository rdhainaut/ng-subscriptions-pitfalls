🧪 Souscriptions Angular : Bonnes et mauvaises pratiques

🌐 Souscriptions orphelines HTTP

📋 Test des souscriptions HTTP orphelines:

  1. Ouvrez la console (F12)
  2. Allez sur "Dangling Call Bad Component" et cliquez "Charger utilisateur"
  3. Naviguez IMMÉDIATEMENT vers "Dangling Call Good Component" (pendant les 3 secondes d'attente)
  4. Console : Dangling Call Bad Component reçoit encore la réponse HTTP après destruction ! 🚨
🔴 Dangling Call Bad Component (avec souscription orpheline) ✅ Dangling Call Good Component (sans souscription orpheline)

⏱️ Fuites mémoire d'intervalles

📋 Test des fuites mémoire d'intervalles:

  1. Allez sur "Interval Bad Component" et cliquez "Démarrer intervalle"
  2. Naviguez vers "Interval Good Component" rapidement
  3. Console : L'intervalle continue de tourner en arrière-plan ! 🚨
  4. Répétez avec "Interval Good Component" : l'intervalle s'arrête automatiquement ✅
🔴 Interval Bad (avec fuite mémoire) ✅ Interval Good (sans fuite mémoire)
💡 Regardez attentivement la console pour voir les messages d'exécution !
